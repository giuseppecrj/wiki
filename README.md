# wiki

An LLM-maintained personal knowledge wiki. Install it as a Claude Code skill, point it at an Obsidian vault or any markdown directory, and let the agent do all the writing, cross-referencing, and maintenance while you curate the sources.

## Install

**Claude Code:**

```
claude skill install pkts.run/wiki
```

**pkts:**

```
pkts install pkts.run/wiki
```

Both install the skill to `~/.claude/skills/wiki/`. Verify with:

```
ls ~/.claude/skills/wiki/SKILL.md
```

If you also have a project-local copy at `.agents/skills/wiki/`, Claude Code will use the project-local version. Global install is the default; project-local overrides it.

## Quick start

Open your vault (or any empty directory) in Claude Code and run:

```
/wiki init
```

The init flow will:
1. Create the directory scaffold (`raw/`, `wiki/`, etc.)
2. Interview you — what the wiki covers, who you are, your topic areas, your preferred writing voice
3. Generate `CLAUDE.md`, `SOUL.md`, `CRITICAL_FACTS.md`, `index.md`, `log.md`, and topic hub pages

Then add your first source:

```
/wiki ingest https://example.com/article
```

Or drop files into `raw/` and run:

```
/wiki ingest
```

Every ingest rewrites existing pages to integrate new knowledge — the wiki gets smarter, not just bigger.

## Commands

| Command | What it does |
| ------- | ------------ |
| `/wiki ingest` | Scan `raw/` for new files, process them all |
| `/wiki ingest [url]` | Fetch and process a specific URL |
| `/wiki query [question]` | Answer a question from the wiki with citations |
| `/wiki health` | Audit: broken links, contradictions, orphans, concept gaps |
| `/wiki synthesize` | Surface unnamed patterns across recent sources |
| `/wiki challenge [idea]` | Red-team an idea against wiki history |
| `/wiki daily` | Create or update today's daily note |
| `/wiki save` | Save conversation insights to the wiki |
| `/wiki reconcile [topic]` | Resolve contradictions between sources |
| `/wiki init` | Bootstrap a new wiki from scratch |

## Architecture

### Three layers

1. **Raw sources** (`raw/`) — your inbox. Drop files here; the agent reads them, processes them into wiki pages, then deletes the originals.
2. **The wiki** (`wiki/`) — agent-owned generated pages. You read them; the agent writes them.
3. **The schema** (`CLAUDE.md`) — tells the agent how the wiki is structured. Generated by init, evolves over time.

### Four page types

| Type | Location | What it holds |
| ---- | -------- | ------------- |
| Source | `wiki/sources/` | One page per ingested source — articles, tweets, PDFs, bookmarks |
| Entity | `wiki/entities/` | People, companies, products, projects |
| Concept | `wiki/concepts/` | Ideas, frameworks, themes, patterns |
| Topic | `wiki/topics/` | Hub pages organizing entities, concepts, and sources by domain |

Plus daily notes at `wiki/daily/YYYY-MM-DD.md`.

### Progressive context loading

The skill loads context in layers so it doesn't read the entire wiki on every operation:

- **L0** (~170 tokens): `SOUL.md`, `CRITICAL_FACTS.md` — who you are, always-true facts
- **L1** (~1-2K tokens): `index.md`, `log.md` (last 10 entries) — what exists, what happened recently
- **L2** (~2-5K tokens): Today's daily note, recent activity — loaded when asking about current state
- **L3** (on demand): Full page reads — loaded only when an operation needs deep context

After L0-L1, the agent knows who you are and what's in the wiki. That's enough to route any operation.

## Philosophy

### Writer, not filing clerk

Every page captures why the subject matters to you — not neutral encyclopedia prose. The wiki is an extension of your thinking, not a database.

### Rewrite, don't append

When a new source adds information about something the wiki already covers, the agent rewrites the existing page to integrate the new knowledge. If an ingest only creates new pages and touches nothing existing, it wasn't deep enough.

### Anti-cramming

If a third paragraph about a sub-topic is being added to an existing page, that sub-topic deserves its own page.

### Anti-thinning

If you can't write at least 3 meaningful sentences about something, don't create the page yet. No stubs, no bookmark shells.

### File system over RAG

The wiki uses `index.md` routing and file navigation — not embeddings or vector databases. Filesystem retrieval doesn't degrade over time. Semantic retrieval does. (This is mathematically proven; see the no-escape theorem.)

### Bi-temporal facts

Track both when something was true and when the wiki learned it, using `## History` sections on entity and concept pages. This enables auditing, reconciliation, and trend detection.

### Voice consistency

During init, you define your writing voice in `SOUL.md`. Every wiki page is written in that voice so the whole wiki sounds like one person wrote it — because the wiki is an extension of you.

## Extending

The wiki skill is designed to be a foundation you build on.

### Adding topic areas

Edit the topic table in your `CLAUDE.md` and create a new hub page at `wiki/topics/<slug>.md` using the template in `pages.md`. The next ingest will start routing relevant material to the new topic.

### Adding page types

The four built-in types (source, entity, concept, topic) handle most needs. If you need a custom type:

1. Add the template to your vault (not to the skill — keep the skill generic)
2. Document it in your `CLAUDE.md`
3. Add a subdirectory under `wiki/` if needed

### Building companion skills

Other skills can feed the wiki by writing to `raw/`:

1. Your companion skill fetches or generates content
2. It writes markdown files to `raw/` following the naming conventions in `conventions.md`
3. The user (or a cron) runs `/wiki ingest` to process them

This is the same pattern used for bookmark importers, tweet archivers, and content pipelines. The wiki skill doesn't need to know about the companion — it just processes whatever lands in `raw/`.

Companion skills can also read from the wiki (query `index.md`, read pages) without modifying it — useful for content generation, briefing, or analysis tools.

### Integrations

The skill gracefully uses optional tools when available:
- **Google Calendar** — daily notes pull today's events
- **YouTube tools** — ingest extracts metadata and subtitles
- **Transcription tools** — ingest transcribes audio sources
- **URL fetching** — ingest processes URLs directly

If a tool isn't available, the skill skips that step silently. No configuration needed.

## Requirements

- [Claude Code](https://claude.ai/code) (any version with skill support)
- An Obsidian vault or any directory where you want markdown files. The skill uses `[[wikilinks]]` for cross-referencing, which Obsidian resolves natively. Other markdown editors that support wikilinks will also work; editors without wikilink support will show the raw syntax but the wiki remains fully functional.
